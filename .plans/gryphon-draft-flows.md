## 3. Draft Lifecycle Flow

```mermaid
stateDiagram-v2
    [*] --> Created: HG creates draft
    Created --> Lobby: HG opens lobby
    Lobby --> Live: HG starts draft (all ready)
    Lobby --> Cancelled: HG cancels
    Live --> Paused: HG pauses
    Paused --> Live: HG resumes
    Paused --> Cancelled: HG cancels
    Live --> Completed: All shifts filled
    Completed --> [*]
    Cancelled --> [*]

    note right of Created
        Sheet connected
        Sub-area selected
        Pre-fills scanned
    end note

    note right of Lobby
        Participants joining
        Ready status tracking
        Re-scan Sheet on start
    end note

    note right of Live
        WebSocket connected
        Picks write to Sheet async
        Timer synchronized
        Emoji reactions
    end note
```

## 11. Real-Time Architecture (WebSocket Events)

```mermaid
sequenceDiagram
    participant G as Gryphon (Client)
    participant S as Server (Rust/Axum)
    participant SH as Google Sheets

    G->>S: connect_ws(draft_id)
    S->>G: draft_state (full sync)

    Note over G,S: During Draft

    G->>S: pick_shift(cell)
    S->>S: Validate (turn, timer, available)
    S->>G: pick_confirmed
    S-->>SH: async write_pick(cell, name)
    S->>G: broadcast: cell_updated

    G->>S: send_reaction(emoji)
    S->>G: broadcast: reaction(emoji, user)

    Note over G,S: Draft Complete

    S->>SH: reconciliation check
    S->>G: draft_completed(results)
```
