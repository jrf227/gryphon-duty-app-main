## US-001: Google OAuth Login

**As a** user (Admin, Head Gryphon, or Gryphon)
**I want to** sign in with my @uwo.ca Google account
**So that** I can access the app with my university credentials and authorize Google Sheets access

### Acceptance Criteria

- [ ] **AC-001.1:** Given the Login screen is displayed, when I click "Continue with Google", then a Google OAuth flow initiates requesting `spreadsheets.edit` and profile scopes
- [ ] **AC-001.2:** Given I authenticate with a valid @uwo.ca account, when OAuth completes, then I am redirected to my role-appropriate dashboard
- [ ] **AC-001.3:** Given I authenticate with a non-@uwo.ca account, when OAuth completes, then an error message is shown and access is denied
- [ ] **AC-001.4:** Given the OAuth flow fails or is cancelled, when I return to the app, then an error alert is displayed with a retry option
- [ ] **AC-001.5:** Given I am already authenticated, when I open the app, then I am taken directly to my dashboard without re-login

### Screen(s): 1. Login (Desktop), Mobile: Dashboard (auto-redirect)
### State(s): default, authenticating, error, authenticated-redirect

## US-003: Import Google Sheet Workbook

**As a** Head Gryphon
**I want to** connect and import my area's existing duty calendar workbook
**So that** the app can read rosters, schedules, and quotas from the Sheet

### Acceptance Criteria

- [ ] **AC-003.1:** Given I am on the Dashboard, when I click to connect a Sheet, then a Google file picker allows me to select a workbook
- [ ] **AC-003.2:** Given I select a valid workbook, when the Sheet Adapter parses it, then it detects sub-areas, block size, and roster ranges automatically
- [ ] **AC-003.3:** Given parsing succeeds, when I see the review screen, then it shows detected sub-areas, roster counts, and week ranges for confirmation
- [ ] **AC-003.4:** Given parsing fails (invalid format), when the error occurs, then a descriptive error message explains what's wrong
- [ ] **AC-003.5:** Given I confirm the import, when it completes, then rosters, pre-filled shifts, and Duty Math quotas are available in the app

### Screen(s): 3. Head Gryphon Dashboard (import flow)
### State(s): not-connected, connecting, parsing, review, confirmed, error

## US-006: Create a Draft

**As a** Head Gryphon
**I want to** create a new draft for a specific sub-area and semester
**So that** Gryphons can pick their shifts in an organized, fair process

### Acceptance Criteria

- [ ] **AC-006.1:** Given I navigate to Drafts, when I see the drafts list, then existing draft cards are displayed in a grid layout

- [ ] **AC-006.5:** Given draft order is configurable, when I want to randomize, then a shuffle button reorders the participant list
- [ ] **AC-006.6:** Given I complete configuration, when I create the draft, then it enters "Created" state and is visible in the drafts list

### Screen(s): 5. Drafts List & Create Draft, Mobile: Commence Draft (Head Gryphon)

## US-008: Participate in Live Draft

**As a** Gryphon
**I want to** pick my preferred shifts on the draft board during my turn
**So that** I can build my ideal schedule within the fair draft system

### Acceptance Criteria

- [ ] **AC-008.1:** Given the draft is Live, when I view the draft board, then I see a 9-column grid (Sun Day–Sat Night) with all weeks for the semester
- [ ] **AC-008.2:** Given it is my turn, when I tap an available cell, then it highlights as selected and I can confirm the pick
- [ ] **AC-008.3:** Given I confirm a pick, when it's processed, then my name and avatar appear in that cell within <200ms (WebSocket broadcast)
- [ ] **AC-008.4:** Given it is not my turn, when I tap a cell, then no selection occurs (cells are view-only)
- [ ] **AC-008.5:** Given a cell is pre-filled, when I view it, then it shows the name with a lock icon and muted styling, and is not selectable
- [ ] **AC-008.6:** Given the top bar is visible, when the draft progresses, then I see: turn indicator ("Round X · [Name]'s Turn"), countdown timer, and progress bar ("X/Y shifts")
- [ ] **AC-008.7:** Given my pick timer expires, when time runs out, then the system handles the timeout per configured behavior (auto-pick/skip)
- [ ] **AC-008.8:** Given the draft uses Snake mode, when Round 1 ends (A→B→C), then Round 2 reverses (C→B→A)

### Screen(s): 7. Live Draft (Desktop), Mobile: Draft Portal (Live), Mobile: Live Draft v2 (Calendar Grid)
### State(s): my-turn, waiting-turn, pick-confirming, paused, completed

## US-013: Propose a Trade

**As a** Gryphon
**I want to** propose trading one of my assigned shifts
**So that** I can swap with another Gryphon for a more convenient schedule

### Acceptance Criteria

- [ ] **AC-013.1:** Given I have assigned shifts, when I navigate to Trade Terminal, then I see a list of my tradeable shifts (only uncompleted ones)
- [ ] **AC-013.2:** Given I select a shift to trade, when I proceed to Step 2, then I can choose: "Broadcast to Trade Board" (public) or "Direct Trade" (specific swap)
- [ ] **AC-013.3:** Given I choose "Broadcast", when I submit, then my shift appears on the Trade Board visible to all Gryphons in my sub-area
- [ ] **AC-013.4:** Given I choose "Direct Trade", when I submit, then a swap proposal is sent to the specific Gryphon
- [ ] **AC-013.5:** Given I have a pending trade, when I want to cancel, then I can cancel before it's accepted

### Screen(s): Mobile: Trade Terminal Step 1, Mobile: Trade Terminal Step 2
### State(s): selecting-shift, choosing-method, broadcasting, direct-proposing, submitted

## US-014: Accept or Decline a Trade

**As a** Gryphon
**I want to** respond to trade proposals directed at me
**So that** I can accept beneficial swaps or decline unwanted ones

### Acceptance Criteria

- [ ] **AC-014.1:** Given I receive a trade proposal, when I view it, then I see: who's proposing, their offered shift, and the shift they want from me

- [ ] **AC-014.5:** Given a trade is not responded to, when the expiry window passes, then it enters Expired state

## US-023: Real-Time Draft State Sync

**As a** Gryphon
**I want to** see other participants' picks appear instantly on my screen
**So that** my view of the draft board is always current

### Acceptance Criteria

- [ ] **AC-023.1:** Given the draft is Live, when another Gryphon makes a pick, then the cell updates on my screen within <200ms via WebSocket
- [ ] **AC-023.2:** Given my WebSocket connection drops, when it reconnects, then the draft state syncs and I see all picks made during disconnection
- [ ] **AC-023.3:** Given the WebSocket connection fails, when reconnection fails, then a reconnection UI overlay is displayed with retry option
- [ ] **AC-023.4:** Given the timer is running, when I compare my timer to other clients, then they are synchronized within 500ms accuracy

### Screen(s): 7. Live Draft, Mobile: Draft Portal, Mobile: Live Draft v2
### State(s): connected, reconnecting, disconnected, synced
