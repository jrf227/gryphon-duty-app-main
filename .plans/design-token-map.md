|----------|-------|---------------|
| `--radius-none` | `0px` | `rounded-none` |
| `--radius-m` | `16px` | `rounded-[16px]` or custom `rounded-m` |
| `--radius-pill` | `999px` | `rounded-full` |

### Tailwind Config Addition

```ts
// tailwind.config.ts
borderRadius: {
  none: '0px',
  m: '16px',       // --radius-m (cards, dropdowns)
  full: '999px',   // --radius-pill (buttons, badges, avatars)
}
```

## Gaps Summary

These variables exist in Pencil but need explicit additions to `tailwind.config.ts`:

| Gap | Pencil Variable | Tailwind Default? | Config Addition |
|-----|-----------------|-------------------|-----------------|
| `--radius-m: 16` | `--radius-m` | No (Tailwind has `rounded-2xl` = 16px) | Use `rounded-2xl` or add `rounded-m` |
| Font: JetBrains Mono | `--font-primary` | No | `fontFamily: { primary: ['JetBrains Mono', 'monospace'] }` |
| Font: Geist | `--font-secondary` | No | `fontFamily: { secondary: ['Geist', 'sans-serif'] }` |

## Notes

- All color variables use **hex format** in the `.pen` file. When implementing with shadcn/ui convention, convert to HSL for CSS custom properties, or use hex directly with Tailwind v4's `@theme` directive.
- The `--sidebar-border` Dark value `#FFFFFF1A` uses alpha transparency — ensure the Tailwind config handles RGBA.
- Button `cornerRadius` uses `--radius-pill` (999px) — all buttons are fully rounded (`rounded-full`).
- Cards use `--radius-none` (0px) — cards have sharp corners in this design system.
